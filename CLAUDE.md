## 1. IMPRESSUM (§5 DDG - Digitale-Dienste-Gesetz)

### Правовая основа
- **§5 DDG** (заменил §5 TMG с 14 мая 2024 года)
- **Штраф за нарушение:** до 50.000 EUR (§33 DDG)

### ОБЯЗАТЕЛЬНЫЕ элементы Impressum
- [ ] Полное имя владельца (имя + фамилия) ИЛИ название компании с правовой формой
- [ ] Полный почтовый адрес (НЕ абонентский ящик!)
- [ ] Email-адрес (обязательно!)
- [ ] Телефон (настоятельно рекомендуется, хотя формально не обязателен по решению EuGH)
- [ ] Правовая форма предприятия (Einzelunternehmen, GmbH, UG и т.д.)
- [ ] Торговый реестр (Handelsregister) - номер и суд регистрации (если есть)
- [ ] USt-IdNr. (Umsatzsteuer-Identifikationsnummer) - если присвоен
- [ ] Wirtschafts-Identifikationsnummer - если применимо
- [ ] Для регулируемых профессий: компетентный надзорный орган с полным адресом
- [ ] Ссылка на платформу ЕС по онлайн-разрешению споров: https://ec.europa.eu/consumers/odr/

### Критические правила
- Impressum должен быть **"leicht erkennbar, unmittelbar erreichbar und ständig verfügbar"** (легко узнаваем, непосредственно доступен и постоянно доступен)
- Максимум **2 клика** с любой страницы сайта до Impressum
- Ссылка "Impressum" в футере на КАЖДОЙ странице
- **ЗАПРЕЩЕНО** ссылаться на "§5 TMG" - только "§5 DDG"!
- Для Einzelunternehmen: обязательно полные имя и фамилия владельца

### Типичные ошибки (ведут к Abmahnung)
- Неполное имя (только имя без фамилии)
- Абонентский ящик вместо физического адреса
- Отсутствие правовой формы
- Отсутствие записи в торговом реестре (если есть)
- Устаревшая ссылка "§5 TMG" вместо "§5 DDG"
- Impressum доступен только через 3+ кликов
- Отсутствие email-адреса

## 3. COOKIE CONSENT (§25 TDDDG)

### Правовая основа
- **§25 TDDDG** (ранее §25 TTDSG)
- **DSGVO Art. 6, 7**
- **Решение EuGH "Planet49"** (C-673/17)
- **Einwilligungsverwaltungs-Verordnung (EinwVO)** - вступила в силу 1 апреля 2025

### ОБЯЗАТЕЛЬНЫЕ требования к Cookie Banner
- [ ] **Активный Opt-In** для всех несущественных cookies (НИКАКИХ предварительно отмеченных галочек!)
- [ ] Первый уровень баннера ОБЯЗАН содержать ОБЕ кнопки: "Alle akzeptieren" И "Alle ablehnen"
- [ ] Обе кнопки **ВИЗУАЛЬНО РАВНЫ**: одинаковый размер, цвет, заметность, количество кликов
- [ ] Детальная информация о каждом сервисе, провайдере и сроке хранения (минимум на втором уровне)
- [ ] Технические cookies (сессия, язык) освобождены ТОЛЬКО если строго необходимы
- [ ] Cookie wall (блокировка контента до согласия) **ЗАПРЕЩЕНА** - согласие не будет "свободным"
- [ ] Согласие должно быть **документировано и доказуемо**
- [ ] Пользователь должен иметь возможность **отозвать согласие так же легко, как дал его**
- [ ] **НИКАКАЯ** передача данных третьим сторонам до получения согласия!

### Критические правила реализации
- Google Analytics НЕ загружается до получения согласия
- Google Maps НЕ загружается до получения согласия (использовать 2-click solution)
- Altegio виджет: проверить, не передаёт ли данные до согласия
- По умолчанию: аналитика ВЫКЛЮЧЕНА
- Настройки cookies доступны повторно (из футера)
- **ЗАПРЕЩЕНЫ Dark Patterns**: кнопка "Отклонить" не должна быть менее заметной

### Типичные ошибки
- Кнопка "Отклонить" скрыта на втором уровне или мельче/другого цвета
- Предварительно отмеченные чекбоксы
- Скрипты загружаются ДО получения согласия
- Нет возможности отозвать согласие позже
- Google Analytics загружается до/без согласия
- Nudging/Dark Patterns (кнопка "Принять" заметнее "Отклонить")

## 4. BARRIEREFREIHEIT / ДОСТУПНОСТЬ (BFSG - Barrierefreiheitsstärkungsgesetz)

### Правовая основа
- **BFSG** (имплементация EU Directive 2019/882 - European Accessibility Act)
- **EN 301 549; WCAG 2.1 Level AA**
- **Дата вступления в силу:** 28 июня 2025
- **Штраф:** до 100.000 EUR

### ВАЖНОЕ ИСКЛЮЧЕНИЕ: Kleinstunternehmen
Предприятия с **менее 10 сотрудниками И максимальным оборотом 2 млн EUR** **ОСВОБОЖДЕНЫ от BFSG для услуг**. Если студия квалифицируется как Kleinstunternehmen, требования BFSG к веб-доступности НЕ применяются.

**ОДНАКО:** Даже при освобождении от BFSG, доступность остаётся лучшей практикой и может стать обязательной по другим законам (AGG - антидискриминационный закон).

### Требования WCAG 2.1 AA (78 критериев успеха)
- [ ] Достаточный контраст цветов: минимум 4.5:1 для обычного текста, 3:1 для крупного
- [ ] Все поля форм корректно промаркированы (label)
- [ ] Структурированная иерархия заголовков (H1, H2, H3...)
- [ ] Навигация клавиатурой для всех интерактивных элементов
- [ ] Alt-текст для всех значимых изображений
- [ ] Субтитры/транскрипты для аудио/видео
- [ ] Масштабируемый текст без потери содержания
- [ ] Информация не передаётся только цветом
- [ ] Видимые индикаторы фокуса
- [ ] Идентификация ошибок и подсказки в формах
- [ ] Единообразная навигация
- [ ] Атрибут lang в HTML-теге
- [ ] Reflow (WCAG 1.4.10): контент при ширине 320px без горизонтальной прокрутки

## 5. PREISANGABENVERORDNUNG (PAngV) - Правила указания цен

### Правовая основа
- **PAngV §§1, 3** (изменена статьёй 6 закона от 3 февраля 2026, BGBl. 2026 I Nr. 28)

### Требования для поставщиков услуг
- [ ] Все цены включают НДС (Bruttopreise/Endpreise)
- [ ] Цены чётко привязаны к конкретным услугам
- [ ] Прейскурант основных услуг должен быть создан
- [ ] Если цены указаны на сайте - только полные цены со всеми обязательными компонентами
- [ ] Указание "inkl. MwSt." при каждой цене
- [ ] Длительность услуги должна быть указана рядом с ценой

### Типичные ошибки
- Показ цен без НДС (netto вместо brutto)
- Отсутствие "inkl. MwSt."
- Цены не привязаны к конкретным услугам
- Скрытые доплаты не включены в цену
- "Ab"-цены (от...) без объяснения, что определяет итоговую цену
- Несоответствие цен на сайте и в студии

### Специфика для beauty-студии
- Цены на лазерные процедуры, маникюр, педикюр - все с НДС
- Пакетные предложения - показывать полную цену пакета
- Подписки (Abonnements) - полная стоимость

## 6. UWG - Закон о недобросовестной конкуренции (Werbung/Реклама)

### Правовая основа
- **UWG §§3, 4, 5, 5a, 6, 7, 8; Приложение к §3 Abs. 3 UWG ("Чёрный список")**
- **EmpCo-Richtlinie (3-е изменение UWG)** - опубликована 19.02.2026, вступает в силу **27.09.2026**

### Требования к рекламе на сайте
- [ ] Никаких вводящих в заблуждение утверждений об услугах (§5 UWG)
- [ ] Никакого пропуска существенной информации (§5a UWG)
- [ ] Никаких агрессивных коммерческих практик (§4a UWG)
- [ ] Сравнительная реклама только по §6 UWG
- [ ] Никаких необоснованных утверждений об эффективности
- [ ] Отзывы должны быть подлинными и проверяемыми

### ЗАПРЕЩЕНО для beauty-студий
- Утверждения о "постоянных" результатах без оговорок
- Суперлативы ("лучший", "самый эффективный") без доказательств
- Медицинские утверждения без доказательств
- Вводящие в заблуждение утверждения о "натуральности" или "органичности"
- Ложные или преувеличенные результаты до/после
- Поддельные или купленные отзывы
- Нераскрытые оплаченные партнёрства с инфлюенсерами

### С 27.09.2026 (EmpCo-Richtlinie)
- **ЗАПРЕЩЕНЫ** общие экологические утверждения: "umweltfreundlich", "ökologisch", "klimafreundlich", "grün" - если не подтверждены признанными экологическими достижениями
- Знаки устойчивости только на основе сертифицированных схем

### Правовая основа
- **§§305-310 BGB**

### Важно

- Если AGB есть → должны соответствовать §§305-310 BGB
- Должны быть чётко представлены и приняты ДО заключения договора
- Неожиданные/несправедливые пункты - ничтожны (§305c BGB)

### Рекомендуемое содержание для beauty-студии
- [ ] Область применения
- [ ] Процесс бронирования и подтверждения
- [ ] Политика отмены (24-часовое уведомление, штраф за неявку)
- [ ] Обязанность клиентов сообщать об аллергиях, состоянии кожи, заболеваниях
- [ ] Письменное согласие родителей/опекунов для несовершеннолетних до 18 лет
- [ ] Ограничения ответственности (в законных пределах)
- [ ] Отсутствие гарантий конкретных результатов лечения
- [ ] Условия оплаты
- [ ] Требования гигиены
- [ ] Политика фото/видео в студии
- [ ] Условия подарочных сертификатов (минимум 3 года действия! §807 BGB)

### Типичные ошибки
- Чрезмерные штрафы за отмену (непропорциональные)
- Полное исключение ответственности
- Скрытые или неожиданные пункты
- AGB недоступны до завершения бронирования
- Противоречия между AGB на сайте и правилами в студии

## 10. WIDERRUFSRECHT (Право отзыва - для онлайн-бронирования)

### Правовая основа
- **§§312b, 312c, 312g, 355-357 BGB; Art. 246a EGBGB**

### Требования
- Онлайн-бронирование beauty-услуг = **Fernabsatzvertrag** (дистанционный договор)
- По умолчанию: **14-дневное право отзыва**
- [ ] Widerrufsbelehrung (инструкция по отзыву) ДО заключения договора
- [ ] Muster-Widerrufsformular (типовая форма отзыва) - Приложение 2 к Art. 246a EGBGB
- [ ] Отправка на долговечном носителе (email) после заключения договора

### Исключение для услуг с конкретной датой (§312g Abs. 2 Nr. 9 BGB)
Beauty-процедуры юридически НЕ однозначно относятся к "Freizeitbetätigungen". **Безопаснее включить Widerrufsbelehrung.**

### Для досрочной отмены права отзыва
1. Получить явное согласие на начало услуги до истечения срока отзыва
2. Получить подтверждение, что клиент знает об утрате права отзыва после полного оказания услуги

### Типичные ошибки
- Полное отсутствие Widerrufsbelehrung при онлайн-бронировании
- Устаревшая форма инструкции
- Утверждение, что право отзыва не существует (когда оно есть)
- Непредоставление типовой формы отзыва
- Неправильный расчёт срока отзыва

## 11. EMAIL-МАРКЕТИНГ / NEWSLETTER

### Правовая основа
- **§7 UWG; Art. 6 Abs. 1 lit. a DSGVO; §25 TDDDG**
- **EuGH C-654/23 (Inteligo Media, ноябрь 2025)** - усилено исключение для существующих клиентов

### Требования
- [ ] **Double Opt-In** - стандарт для подписки на рассылку в Германии
- [ ] Подтверждающее письмо НЕ содержит рекламы!
- [ ] Каждое маркетинговое письмо содержит: Impressum, идентификация отправителя, механизм отписки
- [ ] Ссылка на политику конфиденциальности при форме подписки
- [ ] Согласие документировано (timestamp, IP, клик подтверждения)
- [ ] Согласие не объединено с другими согласиями

### Исключение для существующих клиентов (§7 Abs. 3 UWG)
Email-маркетинг без явного согласия допустим ЕСЛИ:
1. Email получен в связи с продажей/услугой
2. Маркетинг - для собственных аналогичных услуг
3. Клиент не возражал
4. В каждом письме есть возможность отписаться

- **ОСТОРОЖНОСТЬ** с любыми фото до/после для процедур, связанных с изменением тканей

1. **КАЖДЫЙ текстовый элемент** на сайте ОБЯЗАН проходить через функцию `t()` из `useLanguage()` хука

## КРИТИЧЕСКИЕ ПРОБЛЕМЫ ДВУЯЗЫЧНОСТИ (ОБНАРУЖЕНО)

### КРИТИЧЕСКИЙ УРОВЕНЬ

1. **Datenschutz.tsx** - Полный текст политики конфиденциальности **ХАРДКОД НА НЕМЕЦКОМ**
   - Заголовок/подзаголовок через `t()`, но тело документа - хардкод
   - Русскоязычные пользователи видят немецкий юридический текст
   - **Юридический риск:** Нарушение DSGVO (информация должна быть понятна субъекту данных)

2. **Impressum.tsx** - Та же проблема: частичный хардкод на немецком

3. **AGB.tsx** - Вероятно та же проблема

4. **Alt-тексты изображений** - ВСЕ хардкодированы на немецком
   - Пример: `alt="Laser-Haarentfernung München"` вместо `alt={t('image.laser.alt')}`
   - **Влияние:** SEO для русской версии, доступность, BFSG

### СРЕДНИЙ УРОВЕНЬ

5. **Altegio виджет** - Языковая поддержка неясна
   - Виджет загружается глобально без параметра языка
   - Интерфейс бронирования может быть на неправильном языке

6. **JSON-LD Schema** - Некоторые описания хардкодированы на немецком
   - Особенно в LaserHairRemoval.tsx

## ПРАВИЛА ПРИ ДОБАВЛЕНИИ НОВОГО КОНТЕНТА

1. **Никогда** не пишите текст напрямую в JSX. Всегда:
   ```tsx
   // НЕПРАВИЛЬНО ❌
   <h1>Laser-Haarentfernung</h1>

   // ПРАВИЛЬНО ✅
   <h1>{t('laser.title')}</h1>
   ```

2. **Всегда** добавляйте перевод для ОБОИХ языков в LanguageContext.tsx:
   ```tsx
   // В разделе de:
   'laser.title': 'Laser-Haarentfernung',
   // В разделе ru:
   'laser.title': 'Лазерная эпиляция',
   ```

3. **Alt-тексты** изображений - через `t()`:
   ```tsx
   // НЕПРАВИЛЬНО ❌
   <img alt="Laser München" />

   // ПРАВИЛЬНО ✅
   <img alt={t('image.laser.alt')} />
   ```

4. **Форматирование дат и чисел** - учитывать локаль:
   - DE: `01.03.2026`, `1.500,00 €`
   - RU: `01.03.2026`, `1 500,00 €`

5. **Юридические тексты** - перевод ВСЕГО содержания, не только заголовков

# ТРЕБОВАНИЯ К МОБИЛЬНОЙ ВЕРСИИ САЙТА

> **Фреймворк:** Tailwind CSS с стандартными breakpoints
> **Подход:** Mobile-first (Tailwind по умолчанию - мобильный)
> **Viewport:** `<meta name="viewport" content="width=device-width, initial-scale=1.0" />`

---

## BREAKPOINTS

| Breakpoint | Ширина | Применение |
|------------|--------|------------|
| (base) | 0-639px | Мобильные телефоны |
| sm | 640px+ | Большие телефоны / маленькие планшеты |
| md | 768px+ | Планшеты |
| lg | 1024px+ | Ноутбуки |
| xl | 1280px+ | Десктоп |
| 2xl | 1400px+ | Большие экраны |

---

## ОБЩИЕ ПРАВИЛА МОБИЛЬНОЙ ВЕРСИИ

### Обязательные принципы
1. **Mobile-first** подход: базовые стили для мобильных, расширения через breakpoints
2. **Минимум 44x44px** для всех touch-целей (кнопки, ссылки, чекбоксы)
3. **Никакой горизонтальной прокрутки** (кроме таблиц с `overflow-x-auto`)
4. **Читаемый текст** без зума: минимум 16px для body текста
5. **Доступность Cookie Banner и юридических ссылок** на мобильных
6. **Impressum и Datenschutzerklärung** доступны одним кликом с мобильных

### Навигация
- [ ] Hamburger-меню на мобильных (скрыто на md+)
- [ ] Полноэкранный мобильный оверлей с прокруткой
- [ ] Блокировка прокрутки body при открытом меню
- [ ] Закрытие меню при клике на ссылку
- [ ] Touch-целы навигации минимум 44px
- [ ] Language switcher доступен на мобильных

### Контент и макет
- [ ] Одноколоночный макет на мобильных (grid-cols-1)
- [ ] Масштабирование до 2 колонок на md, 4 на lg
- [ ] Padding: `px-4` на мобильных, увеличение на больших экранах
- [ ] Изображения: `w-full` + `object-cover` для сохранения пропорций
- [ ] Таблицы: `overflow-x-auto` для горизонтальной прокрутки данных

### Типографика
- [ ] H1: `text-2xl` на мобильных, до `text-5xl` на lg
- [ ] H2: `text-xl` на мобильных, до `text-4xl` на lg

# ПРАВИЛА РАЗРАБОТКИ (Coding Conventions)

---

## Стек и структура
- **React 18** + **TypeScript 5.8**
- **Vite 5.4** с SSG (Static Site Generation)
- **Tailwind CSS 3.4** + **shadcn/ui** (Radix UI)
- **Supabase** (PostgreSQL + Auth + REST API)
- **React Router DOM 6** для маршрутизации
- **React Hook Form** + **Zod** для форм и валидации

## Файловая структура
```
src/
├── pages/          → Страницы (1 файл = 1 маршрут)
├── components/     → UI-компоненты + Admin-компоненты
├── contexts/       → React Context (Language, Auth)
├── services/       → API-сервисы (Supabase)
├── hooks/          → Кастомные React-хуки
├── lib/            → Утилиты (cookies, supabase client)
├── types/          → TypeScript интерфейсы
├── seo/            → Meta-теги, JSON-LD
└── assets/         → Статические ресурсы
```

## Основные правила
1. **Переводы:** Весь текст через `t()` из `useLanguage()`. НИКОГДА хардкод.
2. **Стилизация:** Только Tailwind CSS классы. Mobile-first.
3. **Компоненты:** shadcn/ui как базовые. Кастомные — в `/components/`.
4. **Формы:** React Hook Form + Zod schema. Валидация на клиенте И сервере.
5. **SEO:** PageHelmet на каждой странице. Meta в `/src/seo/metaTags.ts`.
6. **Маршруты:** Всегда с языковым префиксом `/de/` или `/ru/`.
7. **API-ключи:** Только через `import.meta.env.VITE_*`. НИКОГДА хардкод.
8. **Изображения:** `object-cover`, `loading="lazy"`, alt через `t()`.
9. **Юридические тексты:** Полный перевод на оба языка. Ссылки: DDG (не TMG), TDDDG (не TTDSG).
